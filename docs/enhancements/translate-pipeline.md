# 英訳パイプラインの完成

ステータス: **未着手**

**学習エコシステム(`docs/learning-ecosystem/`)とは無関係の、既存ブログ機能に対する
独立したエンハンスメント**。学習サイクル(捕獲・消化・執筆・定着)の設計とは関係なく、
公開済み記事を英語圏向けの資産に変換する機能強化として、別タイミング・別判断で着手する。

## 現状

`.github/workflows/translate.yml` は「`content/blog/*.md` の変更を検知して英訳し
`content/blog/en/` へコミットする」という**構想の骨組みだけ**が存在するスタブ。

- 変更ファイル検知ステップ(`git diff` で差分抽出)は動作する
- 翻訳ステップは `echo "Translating changed files: ..."` と表示するだけで、
  実際の翻訳処理は未実装
- 英訳ファイルをコミットするステップはコメントアウトされたまま無効化されている

つまり現状、日本語記事を push しても英語版は自動生成されない。

フロント側(`src/pages/blog/[slug].tsx`)は `content/blog/en/<slug>.md` が存在すれば
それを英語表示に使うフォールバックロジックが**実装済み**。存在しなければ日本語版を表示し、
「英訳がまだない」旨のバナーを出す。つまり**手動で英訳ファイルを置けば表示自体は動く**。
自動化だけが欠けている状態。

## 完成させるための作業

1. 翻訳ステップの実装(候補: Anthropic API を呼んで frontmatter を保ったまま本文を翻訳)
2. コメントアウトされているコミットステップの有効化
3. 翻訳API呼び出しのシークレット(`ANTHROPIC_API_KEY` 等)の設定
4. 翻訳品質・frontmatter(title/summary/tags等)の扱い方針の確定
   (タイトルも訳すか、tagsは英語表記に変換するか等)
5. 誤訳・機械翻訳感のレビュー方法(全自動でコミットしてよいか、PRを経由すべきか)

## 位置づけ

学習エコシステムのロードマップ([../learning-ecosystem/](../learning-ecosystem/README.md))には
含めない。着手する場合はこのファイルを起点に、別途仕様を詰めてから実装する。
