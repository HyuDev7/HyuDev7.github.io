# Learning Ecosystem ドキュメント

このディレクトリは、本リポジトリ(個人ポートフォリオ + 技術ブログ)の上に構築する
**個人学習エコシステム**の設計ドキュメント一式である。

## このドキュメントの目的

1. **引き継ぎ可能性** — プロジェクトオーナー以外の人間が読んでも、構想・現状・残作業を
   把握して開発を引き継げること。
2. **AIエージェントへのコンテキスト供給** — AIエージェント(Claude Code 等)に実装を
   依頼する際、このディレクトリを読ませるだけで、誤解なく作業に着手できること。

したがって、**実装がこの設計から乖離した場合は、実装と同じPR/コミットでこのドキュメントを
更新すること**。ドキュメントが現実と食い違った時点で上記の目的は失われる。

## 30秒サマリ

- このサイトは Next.js Static Export + GitHub Pages 製。`content/blog/*.md` を push すると
  GitHub Actions が自動公開する仕組みが**既に動いている**。
- その上に「**キャッチアップ(捕獲) → 消化 → 執筆(定着) → 資産化**」の学習サイクルを
  回すためのレイヤーを追加する。
- 核になるのは軽量な **TILノート層**(`content/notes/`)と、**スマホから GitHub Issue 経由で
  ノートを投稿できる捕獲経路**。オーナーのインプットは移動中・書籍・動画が大半のため、
  スマホ捕獲がエコシステムの心臓部である。
- 記事執筆ペースの想定は**月1〜2本 + 日々のノート**。システムの重心はノート側に置く。
- AIの関与は「翻訳・統計・傾向コメント・記事骨子の提案」まで。**記事本文は必ず人間が書く**
  (執筆行為そのものが知識定着の手段であるため)。

## 用語集

| 用語 | 意味 |
|------|------|
| ノート / TILノート | `content/notes/` に置く断片的な学習メモ。5分で書ける粒度。生煮えのまま公開する |
| 記事 / 投稿 | `content/blog/` に置く完成度の高いブログ記事。既存の仕組み |
| 捕獲 (Capture) | 学んだことをノートとして記録に残す行為。L0 |
| 消化 (Digest) | 溜まったノートを素材にテーマを咀嚼し、記事の形に再構成する行為。L1 |
| 昇格 | あるタグのノート群を1本の記事にすること(L0 → L2) |
| パイプライン | タグ単位で見た「ノート在庫 → 記事化済み」の流れ。/learning で可視化する |
| 週次サマリ | 週1回 GitHub Actions が起票する統計 + 傾向コメントの Issue |
